Made in a one week jam in CMPM170, Game Design Studio at UCSC, Future In Flames is a supernatural survival game about being lost in the woods, with only a candle and a knack for predestination to guide you. My role in the team was a Programmer. For my part in development, I programmed an event system which used pre-set dialogue, and a variable amount of options to pick for paths to take. When a player picks a path, other game values can be changed with each choice of an event as a reaction (You found apples along the left path, +10 Food!) I also created the volumetric and post processing fog effects for the visual aspect of the game

  • Data structures/Systems

Music Is The Way

Made for CMPM151, Generative Audio for Games, me and my team created an endless 3d platformer with procedurally generated audio is linked to the position of the platforms. While traversing each level, you are able to toggle an instrument to allow you to jump to certain platforms. As the teams programmer, I created the Unity scene, the movement, the platform placing, the FFT reactive bloom, and linked the Pure Data to the Unity Project.

  • Procedural Generation

Zombie Colosseum

The first group jam game made for CMPM170 at UCSC. Zombie Colosseum was a group project where the goal was to make the audio for the game interactive. Using FMOD and Unity, we created a survival game where the background music is reactive to how many kills you get while surviving. I worked to help implement the FMOD audio functions into the game, as well as the gameplay programming like the monster spawning, moving, interactions, ect.
